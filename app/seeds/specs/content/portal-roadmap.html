Radaptor Roadmap

What already works, what is shipping now, and what comes next

This page is intentionally concrete. It shows the current demo surface, the work happening right now, and the next visible capabilities we want to put on top of the platform.

Done

Registry-first package flow is stable

The Portal and the main app now run on the cleaned-up dev versus registry split. The destructive package update bug is fixed, the nested package repositories are restored, and the install/update flow no longer destroys working checkouts.

Done

Portal public surface is live

The homepage, comparison page, access-request flow, login, and admin entry points already show the kind of product shell Radaptor can expose from one application model.

Publishing now

Tech demo hardening and Portal showcase pass

We are tightening the repo baseline, moving the active apps to PHP 8.5, and keeping the current demo surface stable enough to publish as a first public technical showcase.

Publishing now

Image pipeline cleanup

The immediate goal is a behavior-preserving cleanup of the existing resize, crop, thumbnail, and mask flow. This is about making the current image handling easier to trust, not replacing it with a brand-new platform before the demo.

Planned

Drag and drop page editor

A visual editor with widget palette, canvas, and property editing is the next major usability step. It is deliberately out of the first demo scope so the public release stays focused.

Planned

Deeper publish-readiness pass

After the tech demo is out, the next iteration covers the formal release work: security pass, license audit, versioning discipline, and the more polished operational packaging story.
